          系统结构设计 #

          一. 系统总体结构图 #

          二. 系统 #

          目标：构建一个千变万化的系统结构。 首先系统和目录是虚拟的，可以随意构造。通过选取功能库中的功能，来建立不同的系统。

          系统在数据表存储为一张单表,单表必须存在下面相关字段:

          1. id : 序号
          2. name : 系统名称
          3. comment : 描述

          三. 目录(功能) #

          功能菜单不在是之前的三级权限，功能可以停止在任意目录级别, 根据type进行类别判断。当type为func时，此目录为功能层。

          目录（功能），在数据存储结构上也是一张单表,单表必须存在下面相关字段:

          1. id : 序号
          2. system_id : 系统id
          3. name: 目录名称
          4. par_id : 上级目录id
          5. type : 目录类别 （dir-目录、func-功能）
          6. comment : 功能单元描述.
